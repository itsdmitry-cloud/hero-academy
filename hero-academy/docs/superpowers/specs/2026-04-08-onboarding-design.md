# Онбординг — Спецификация дизайна

**Дата:** 2026-04-08
**Статус:** Утверждён

## Обзор

Свайпируемые онбординг-страницы для мобильного PWA Hero Academy. Показываются при первом входе нового ученика и при демо-входе. Одинаковый набор слайдов для обоих сценариев.

## Решения

| Параметр | Решение |
|----------|---------|
| Количество слайдов | 7 |
| Тон | Микс: эмоциональные начало/конец + механики в середине |
| Визуальный стиль | Скриншоты реального UI с подписями |
| Навигация | Свайп пальцем + точки-индикаторы |
| Элементы управления | «Пропустить» (слайды 1-6), «Далее», «Начать приключение» (слайд 7) |
| Различие демо/новый | Нет — одинаковый онбординг |

## UX-паттерны

### Навигация между слайдами

- **Свайп:** Горизонтальный свайп пальцем влево/вправо
- **Точки-индикаторы:** 7 точек внизу экрана
  - Неактивные: `8×8px`, `rgba(255,255,255,0.2)`, `border-radius: 4px`
  - Активная: `24×8px` (вытянутая пилюля), цвет зависит от слайда:
    - Слайды 1-4, 6: `#a855f7` (фиолетовый)
    - Слайд 5: `#ffd700` (золотой — Королевский Набор)
    - Слайд 7: `#2ed573` (зелёный — финал)
- **«Пропустить»:** Справа вверху нижней панели, слайды 1-6. Цвет: `#6b6b8d`. По нажатию → `/hero`
- **«Далее»:** Кнопка внизу, фиолетовый градиент, слайды 1-6
- **«Начать приключение»:** Зелёная кнопка на слайде 7

### Триггеры показа

| Сценарий | Триггер | Действие |
|----------|---------|----------|
| Новый ученик | Первый вход после регистрации через код класса | Показать онбординг перед `/hero` |
| Демо вход | После успешного `/api/demo/setup` | Показать онбординг перед `/hero` |
| Повторный вход | Ученик уже видел онбординг | НЕ показывать (флаг в localStorage или БД) |

### Флаг прохождения

- Сохранять `onboarding_completed: true` в `localStorage` (ключ: `hero-academy-onboarding`)
- Устанавливать при:
  - Нажатии «Начать приключение» (слайд 7)
  - Нажатии «Пропустить» (любой слайд)
- Проверять при маршрутизации в middleware или layout студента

## Технические решения

### Компонент

Создать `src/components/onboarding/OnboardingCarousel.tsx` + `OnboardingCarousel.module.css`:
- CSS-модуль (как все компоненты проекта — без Tailwind)
- Свайп через `touch-action: pan-y` + `touchstart/touchmove/touchend` события (без внешних библиотек)
- CSS `transform: translateX()` с `transition: transform 300ms ease-out` для анимации
- 7 дочерних компонентов-слайдов: `OnboardingSlide1.tsx` ... `OnboardingSlide7.tsx`

### Маршрутизация

- Новая страница: `src/app/onboarding/page.tsx`
- Без BottomTabBar (вне `(student)` layout)
- Редиректы:
  - После регистрации (`/auth/join`) → `/onboarding`
  - После демо (`/api/demo/setup`) → `/onboarding`
  - После онбординга → `/hero`
  - Если `onboarding_completed` → пропуск, сразу `/hero`

### Анимации

- Свайп: `transform: translateX()`, `transition: 300ms ease-out`
- Точки: `width` transition `200ms ease`
- Кнопка CTA (слайд 7): `scale(1.02)` пульс каждые 2 секунды
- Частицы: CSS `@keyframes float` — медленное перемещение вверх/вниз (4-6с цикл)
- `prefers-reduced-motion`: отключить частицы и пульс, оставить свайп

### Адаптивность

- Mobile-first: оптимизировано под 375-430px ширины
- Контент масштабируется через `rem` и `%`
- `min-height: 100dvh` для каждого слайда
- Safe area: `padding-bottom: env(safe-area-inset-bottom)` для кнопок
