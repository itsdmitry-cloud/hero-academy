# 07. Механика Сезонных Боссов и Классовых Аур

Документация отражает актуальную (v1.5) структуру рейдовых боссов по предметам, систему расчёта урона и командного взаимодействия (Ауры).

## 1. Архитектура Сезонных Боссов (`subject_bosses`)

Вместо ручного создания боссов учителями, механика работает следующим образом:
- При активации нового Сезона (через Админ-панель), система **автоматически генерирует** по одному боссу для каждого предмета, который преподается в классе. 
- Эти боссы хранятся в таблице `subject_bosses`. У босса формируется большой объём здоровья (HP), зависящий от множителя `boss_hp_multiplier` в настройках Админки (Class -> School -> Global).
- **Живучесть босса:** Босс живет всю четверть (сезон), чтобы его хватало на длительное время.

### Видимость
- **Учитель:** Видит всех боссов класса на своем дашборде.
- **Ученик:** Видит боссов *только тех предметов*, которые закреплены за учеником в БД (`profile.subjects`). Если список предметов пуст (по умолчанию), ученик участвует в битвах и видит боссов по **всем** предметам класса.

## 2. Механика боя (Урон и Контратака)

* Ученики "бьют" босса своей активностью (правильными ответами на Контрольной и Домашних заданиях).
* Если ученик получает плохую оценку (ошибается) — Босс **контратакует** и наносит урон самому ученику (снижает его личное HP).

### Декаплинг Урона (Разделение Опыта и Урона)
Ранее урон по боссу жестко привязывался к финально заработанному XP. Это приводило к инфляции: если герой выпивал Личные зелья на XP, он наносил несоразмерный урон боссу.

**Новая математика урона:**
1. **Базовый Опыт (Base XP):** Получается за выполнение задания (например, 100 XP за '5').
2. Урон по боссу рассчитывается **ИСТИННО от Базового Опыта**, игнорируя личные XP-артефакты (бусты опыта).
3. На урон влияют только специализированные модификаторы урона и общие Ауры.

```
Формула: Урон = Base XP * (1 + (Personal_Boss_Dmg_Boost / 100)) * (1 + (Class_Boss_Dmg_Aura / 100))
```

## 3. Классовые Ауры (Командная Работа)

В игру внедрена уникальная механика сплоченности: **Модификаторы окружения**. Система (`artifact-engine.ts`) сканирует инвентари *всех одноклассников* на наличие активных "Командных Аур" перед каждым расчетом.

**Кэширование:** Поскольку сканирование инвентарей всего класса создает огромную нагрузку на БД (особенно при пакетном выставлении 30 оценок), реализовано In-Memory кэширование (5 секунд) для аур класса.

**Пример:**
- **Эликсир Огненного Натиска (Rare):** Даёт +20% к урону по боссам для **всего класса** на 24 часа. Достаточно одному ученику активировать бафф, и урон всех одноклассников в течение суток будет умножаться. (Отображается в логах как: `⚔️ Урон Боссу: 120 (Включает Ауру +20%)`).

## 4. Разделение Лута (Loot Grading)

Если класс убивает босса (Суммарный урон учеников >= HP босса), выдается коллективная награда (Лутбокс), но её Тир строго зависит от личного вклада:

1. **Базовый Тир Сундука** определяется текущим уровенем Ученика:
    * Lv 1-15: 📦 Common / Lv 16-30: 🎁 Rare / Lv 31-45: 💎 Epic / Lv 46+: 👑 Legendary
2. **Бонусы Вклада (Damage Contributions):**
    * Нанес 0% урона (не участвовал): Штраф **-1 Тир** (Наказание за безделье).
    * Нанес > 25% от HP босса: Бонус **+1 Тир**.
    * Нанес максимальный в классе урон (MVP): Доп. Бонус **+1 Тир**.
3. *Ограничение:* Финальный Тир никогда не превышает 👑 Legendary.
