# 01. Архив: Инвентарь и Лутбоксы

## Реестр Артефактов (Single Source of Truth)
Файл: `src/lib/game/artifact-registry.ts`
Любое добавление нового предмета происходит **исключительно** через этот реестр. База данных просто следует за ним.

## Правило Иконок (Strict UI Path)
* Запрещено использовать Emoji в качестве значений поля `icon` в реестре (например, `📦`).
* Все иконки должны быть относительными путями к физическим ресурсам:
  * Стандартные артефакты: `/assets/artifacts/item_key.png`
  * Сундуки: `/assets/lootboxes/rarity.png` 

## Механика Сундуков (Лутбоксы)
* Идентифицируются по `effect_type: 'lootbox'`.
* **ЗАПРЕТ НА СТАКИНГ:** В таблице `hero_artifacts` сундуки **запрещено** выдавать с `quantity > 1`. 
* Обоснование: API `/api/game/open-lootbox` использует удаление записи по её уникальному идентификатору `hero_artifact_id`. Если запихнуть 5 сундуков в одну ячейку, открытие одного сотрёт все 5 сундуков. Выдавать каждый сундук отдельной строкой с `quantity: 1`.

## Архитектурное правило: Определение сезонных сундуков
Сезонные сундуки ДОЛЖНЫ идентифицироваться на бэкенде исключительно по колонке `season_pool` из таблицы базы данных `artifacts`, а не по параметру `boxRarity`, который присылает клиент.
Обоснование: Сезонные сундуки могут иметь стандартную редкость (например `rare`), которая используется для UI-раскраски, но она не должна сбивать логику выдачи сезонного лута. API обязано делать Join с таблицей `artifacts` и проверять наличие `season_pool`.

## Архитектурное правило: UI-Обертка для Зелий-Баффов
Зелья и эликсиры с продолжительным действием (например, XP Combo или Gold Boost) на уровне Базы Данных обязаны парситься движком как **Экипировка** (`is_equipped = true`), так как `artifact-engine.ts` сканирует "Полку героя" после срабатывания триггеров квеста.
ОДНАКО, на уровне Frontend (`inventory/page.tsx`) они должны маскироваться под расходуемые предметы, чтобы не нарушать ролевое погружение (immersion):
- Кнопка экипировки переименовывается в *"✨ Выпить эликсир"*.
- Бейдж экипировки визуализируется как *"✨ АКТИВНО"* (а не *"НА ПОЛКЕ"*).
- Кнопка *"Снять с полки"* (Unequip / Отмена) **принудительно блокируется/скрывается** — выпитое зелье нельзя "отменить" или вернуть в рюкзак, оно сгорает только после завершения квеста.
